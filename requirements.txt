# WiGuard Nexus v5.16.1 — Professional Hardening Release

WiGuard Nexus is a cybersecurity/network evidence analysis platform for importing network configuration evidence, Packet Tracer exports, wireless/security data, and producing normalized assets, evidence-backed findings, topology context, and professional reports.

## v5.16.1 professional hardening highlights

- Added a dedicated parser registry and import validator for JSON, XML, CSV, text/config files, safe ZIP evidence bundles, and Packet Tracer native hooks.
- Added a stable normalized schema for devices, interfaces, IP addresses, VLANs, routes, wireless networks, security rules, topology links, findings, and evidence rows.
- Added a conservative risk engine that avoids fake Critical severity and produces findings with evidence, confidence, impact, and recommendations.
- Added a System Health page plus `/api/system-health` and `/api/normalized-data`.
- Added redacted Professional JSON/HTML report exports with HTML escaping for imported evidence.
- Added internal tools for dependency checking, preflight checks, sample demo data generation, and safe cleanup.

## v5.16.1 hardening notes

- ZIP imports are handled as safe evidence bundles: path traversal members are skipped, unsupported files are ignored with warnings, and native `.pkt/.pka` members are manifest/hash evidence only.
- Friendly validation errors avoid exposing absolute local paths to normal UI users.
- Report HTML now escapes imported evidence before rendering.
- Runtime time helpers use timezone-aware UTC timestamps to avoid Python 3.13+ deprecation warnings.
- Release hygiene now includes `.gitkeep` placeholders for runtime data folders and ignores generated artifacts.

## Supported import formats

JSON, XML, CSV, TXT/LOG/CFG/CONF network configs, ZIP bundles containing supported evidence, and `.pkt`/`.pka` native Packet Tracer files as best-effort native evidence. For full technical claims, prefer exported running-config/show-command/XML/JSON evidence.

## Important limitations

Native Packet Tracer `.pkt` / `.pka` files are proprietary. WiGuard can store, hash, inspect, bridge, and expose recoverable evidence, but it does not fake full topology extraction when the binary does not expose it. Upload companion exported evidence for stronger results.

### v5.8.9 Packet Tracer Fidelity + UI Cleanup

`.pkt/.pka` uploads now run automatically through the full background conversion path:

1. preserve the original file and calculate SHA-256,
2. probe `PTEXPLORER_PATH` or another configured converter if available,
3. recover embedded ZIP/XML/JSON/config text, wrapped streams, zlib chunks, and visible Cisco-like evidence,
4. generate `internal_pkt_bridge.xml`,
5. generate `internal_pkt_bridge.normalized.json`,
6. parse the bridge and decoded payloads into devices, interfaces, VLANs, links, IP inventory, policy evidence, and report artifacts.

The UI exposes `Background Auto-Conversion Pipeline` and `Decoded Native Payloads` so the upload result is transparent instead of a vague low understanding percentage.

## Best accuracy workflow

Native Packet Tracer files are proprietary. WiGuard can safely store `.pkt/.pka` files and recover printable evidence, but the most accurate workflow is:

1. Open the Packet Tracer lab.
2. Export each device running-config as text.
3. Add these command outputs where possible:
   - `show running-config`
   - `show vlan brief`
   - `show interfaces trunk`
   - `show ip interface brief`
   - `show access-lists`
   - `show cdp neighbors detail`
   - `show spanning-tree`
   - `show port-security interface`
   - `show etherchannel summary`
   - `show mac address-table`
   - `show arp`
4. Put the files into one ZIP.
5. Upload the ZIP in the Import Center.
6. Review Conversion Readiness and Missing Command Checklist.
7. Generate reports and Evidence ZIP.
