Policy Precision Cookbook

new rule logic patterns
Evidence Required

Every strong rule should name the exact command or data source needed. Example: show interfaces trunk + show vlan brief.

False Positive Guard

Rules should explain when to keep a result as Review instead of Fail. Missing operational evidence is not the same as a confirmed failure.

Acceptance Criteria

Write the proof needed to close the issue: expected VLAN present, ACL applied to interface, counter evidence captured, or risk accepted.

Runtime Proof

Use ACL hit counters, interface counters, CDP/LLDP, and route table evidence to avoid relying only on planned configuration.
